⚡ Módulo 02

useReducer

useReducer resolve um problema diferente do useState: quando a lógica de atualização de estado é complexa o suficiente para merecer ser isolada, testada e nomeada. O padrão que Redux popularizou, agora nativo e sem dependências.

01

O que é um reducer

Um reducer é uma função pura que recebe o estado atual e uma ação, e retorna o novo estado. Pura significa que, para os mesmos inputs, sempre produz o mesmo output — sem efeitos colaterais, sem mutação direta.

A assinatura de um reducer é sempre a mesma: (state, action) => newState. O estado nunca é mutado diretamente — a função retorna um novo objeto. Isso torna as transições de estado previsíveis e auditáveis.

→

→
UI

O fluxo é unidirecional e explícito. Cada mudança de estado tem um nome (o tipo da action) e uma causa documentada. Isso é o que torna o useReducer superior ao useState quando o estado é complexo: a lógica de transição fica centralizada, testável e legível.

🧠
Conexão com Redux: Redux inteiro é construído sobre esse padrão. Se você entender useReducer, você entende Redux. A diferença é que Redux adiciona uma store global, middleware e ferramentas de DevTools por cima. Para estado local ou semi-global, useReducer resolve com muito menos complexidade.
02

Anatomia completa

Quatro conceitos formam o vocabulary completo do useReducer. Dominar cada um separadamente é o caminho para usá-los juntos sem confusão.

Glossário do useReducer
state
O dado atual. Um objeto (ou primitivo) que representa o estado completo daquele pedaço da UI. É sempre imutável — nunca modificado diretamente.
action
Um objeto que descreve o que aconteceu. Tem um campo type (string que nomeia o evento) e opcionalmente um payload com os dados necessários para a mudança.
dispatch
Função retornada pelo useReducer. Chame-a com uma action para disparar uma transição de estado. É estável entre renders — não precisa de useCallback.
reducer
A função pura que recebe (state, action) e retorna o novo estado. Normalmente usa switch no type da action. É definida fora do componente para ser testável e estável.
initialState
O estado inicial passado como segundo argumento do useReducer. Pode ser um objeto literal ou uma função (terceiro argumento) para inicialização preguiçosa.
anatomia-visual.tsx 
// ─── 1. TIPOS ───────────────────────────────────────────────
type State = { count: number }

// União discriminada — TypeScript infere o payload pelo type
type Action =
  | { type: 'INCREMENT' }
  | { type: 'DECREMENT' }
  | { type: 'RESET' }
  | { type: 'SET'; payload: number }

// ─── 2. REDUCER (fora do componente!) ───────────────────────
function reducer(state: State, action: Action): State {
  switch (action.type) {
    case 'INCREMENT':
      return { count: state.count + 1 }
    case 'DECREMENT':
      return { count: state.count - 1 }
    case 'RESET':
      return { count: 0 }
    case 'SET':
      return { count: action.payload }    // payload só existe neste case
    default:
      return state                         // nunca muta, sempre retorna estado
  }
}

// ─── 3. USO NO COMPONENTE ────────────────────────────────────
function Counter() {
  const [state, dispatch] = useReducer(reducer, { count: 0 })
  //           ^^^^^^^^
  //           função que envia actions ao reducer

  return (
    <div>
      <p>{state.count}</p>
      <button onClick={() => dispatch({ type: 'INCREMENT' })}>+</button>
      <button onClick={() => dispatch({ type: 'DECREMENT' })}>-</button>
      <button onClick={() => dispatch({ type: 'RESET' })}>Reset</button>
      <button onClick={() => dispatch({ type: 'SET', payload: 10 })}>Ir para 10</button>
    </div>
  )
}
💡
União discriminada: quando type é 'SET', TypeScript sabe automaticamente que payload existe e é number. Nos outros cases, payload sequer existe no tipo. Isso elimina a necessidade de verificações manuais e erros em tempo de compilação.
03

Reducer vs useState

Ambos gerenciam estado local. A escolha entre eles depende da complexidade da lógica de atualização e de quantos campos de estado precisam mudar de forma coordenada.

useState
  • Estado simples: string, number, boolean
  • Atualizações independentes entre si
  • Lógica cabe em uma linha
  • Poucas transições de estado
  • Evitar quando 3+ campos mudam juntos
  • Evitar quando a lógica de update é complexa
useReducer
  • Estado com múltiplos campos relacionados
  • Transições nomeadas e auditáveis
  • Lógica complexa ou condicional
  • Próximo evento depende do estado atual
  • Lógica precisa ser testada isoladamente
  • Overhead para estado simples
Situação useState useReducer
toggle: true/false ✓ ideal overhead
input de formulário simples ✓ ideal overhead
Formulário com 5+ campos e validação 5 useState separados ✓ ideal
Lista com CRUD (add, edit, remove) lógica no componente ✓ ideal
Estado de requisição (loading/data/error) 3 useState relacionados ✓ ideal
Máquina de estados (wizard, stepper) ✗ propenso a bugs ✓ ideal
04

Passo a passo com TypeScript

O processo de construir um reducer segue sempre a mesma sequência. Definir os tipos antes de escrever a lógica é o que torna o TypeScript útil aqui — os erros aparecem na definição, não em runtime.

1
Modele o estado com uma interface
Agrupe todos os campos relacionados em um único objeto. Isso permite que o reducer substitua múltiplos campos em uma única ação.
taskReducer.ts — Passo 1 
// Estado: tudo que o componente precisa saber
interface Task {
  id: string
  title: string
  done: boolean
}

interface TaskState {
  tasks: Task[]
  filter: 'all' | 'active' | 'done'
  editingId: string | null
}
2
Defina as actions como união discriminada
Cada action representa um evento que pode acontecer. O nome deve ser descritivo do que aconteceu, no passado — como um log de eventos.
taskReducer.ts — Passo 2 
type TaskAction =
  | { type: 'TASK_ADDED'; payload: { title: string } }
  | { type: 'TASK_TOGGLED'; payload: { id: string } }
  | { type: 'TASK_REMOVED'; payload: { id: string } }
  | { type: 'TASK_EDIT_STARTED'; payload: { id: string } }
  | { type: 'TASK_EDIT_SAVED'; payload: { id: string; title: string } }
  | { type: 'FILTER_CHANGED'; payload: { filter: TaskState['filter'] } }
3
Escreva o reducer
Cada case retorna um novo objeto de estado. Nunca modifique state diretamente — use spread para copiar e atualizar.
taskReducer.ts — Passo 3 
const initialState: TaskState = {
  tasks: [],
  filter: 'all',
  editingId: null,
}

function taskReducer(state: TaskState, action: TaskAction): TaskState {
  switch (action.type) {

    case 'TASK_ADDED':
      return {
        ...state,
        tasks: [
          ...state.tasks,
          { id: crypto.randomUUID(), title: action.payload.title, done: false },
        ],
      }

    case 'TASK_TOGGLED':
      return {
        ...state,
        tasks: state.tasks.map(task =>
          task.id === action.payload.id
            ? { ...task, done: !task.done }
            : task
        ),
      }

    case 'TASK_REMOVED':
      return {
        ...state,
        tasks: state.tasks.filter(task => task.id !== action.payload.id),
      }

    case 'TASK_EDIT_STARTED':
      return { ...state, editingId: action.payload.id }

    case 'TASK_EDIT_SAVED':
      return {
        ...state,
        editingId: null,
        tasks: state.tasks.map(task =>
          task.id === action.payload.id
            ? { ...task, title: action.payload.title }
            : task
        ),
      }

    case 'FILTER_CHANGED':
      return { ...state, filter: action.payload.filter }

    default:
      return state
  }
}
4
Use no componente
O componente fica limpo: apenas rendering e chamadas de dispatch. Toda a lógica de estado vive no reducer.
TaskList.tsx — Passo 4 
import { useReducer, useState } from 'react'
import { taskReducer, initialState } from './taskReducer'

export function TaskList() {
  const [state, dispatch] = useReducer(taskReducer, initialState)
  const [newTitle, setNewTitle] = useState('')

  const addTask = () => {
    if (!newTitle.trim()) return
    dispatch({ type: 'TASK_ADDED', payload: { title: newTitle.trim() } })
    setNewTitle('')
  }

  // Filtragem acontece aqui — é lógica de rendering, não de estado
  const visibleTasks = state.tasks.filter(task => {
    if (state.filter === 'active') return !task.done
    if (state.filter === 'done') return task.done
    return true
  })

  return (
    <div>
      <input
        value={newTitle}
        onChange={e => setNewTitle(e.target.value)}
        placeholder="Nova tarefa..."
      />
      <button onClick={addTask}>Adicionar</button>

      <ul>
        {visibleTasks.map(task => (
          <li key={task.id}>
            <input
              type="checkbox"
              checked={task.done}
              onChange={() => dispatch({ type: 'TASK_TOGGLED', payload: { id: task.id } })}
            />
            <span style={{ textDecoration: task.done ? 'line-through' : 'none' }}>
              {task.title}
            </span>
            <button onClick={() => dispatch({ type: 'TASK_REMOVED', payload: { id: task.id } })}>
              ✕
            </button>
          </li>
        ))}
      </ul>
    </div>
  )
}

Exemplo 1 — Contador avançado

Um contador com múltiplas operações demonstra o padrão mais simples de useReducer. Observe como o componente não tem nenhuma lógica — apenas rendering e dispatch.

src/components/AdvancedCounter.tsx 
import { useReducer } from 'react'

// ─── Tipos ───────────────────────────────────────────────
interface CounterState {
  count: number
  step: number
  history: number[]
}

type CounterAction =
  | { type: 'INCREMENT' }
  | { type: 'DECREMENT' }
  | { type: 'RESET' }
  | { type: 'SET_STEP'; payload: number }
  | { type: 'UNDO' }

// ─── Reducer ─────────────────────────────────────────────
const initialState: CounterState = { count: 0, step: 1, history: [] }

function counterReducer(state: CounterState, action: CounterAction): CounterState {
  switch (action.type) {

    case 'INCREMENT':
      return {
        ...state,
        count: state.count + state.step,
        history: [...state.history, state.count],
      }

    case 'DECREMENT':
      return {
        ...state,
        count: state.count - state.step,
        history: [...state.history, state.count],
      }

    case 'RESET':
      return initialState

    case 'SET_STEP':
      return { ...state, step: action.payload }

    case 'UNDO':
      if (state.history.length === 0) return state
      const previous = state.history[state.history.length - 1]
      return {
        ...state,
        count: previous,
        history: state.history.slice(0, -1),
      }

    default:
      return state
  }
}

// ─── Componente ──────────────────────────────────────────
export function AdvancedCounter() {
  const [state, dispatch] = useReducer(counterReducer, initialState)

  return (
    <div>
      <h2>{state.count}</h2>
      <p>Step: {state.step}</p>

      <button onClick={() => dispatch({ type: 'DECREMENT' })}>
        -{state.step}
      </button>
      <button onClick={() => dispatch({ type: 'INCREMENT' })}>
        +{state.step}
      </button>
      <button onClick={() => dispatch({ type: 'UNDO' })} disabled={state.history.length === 0}>
        ↩ Desfazer
      </button>
      <button onClick={() => dispatch({ type: 'RESET' })}>
        Reset
      </button>

      <label>
        Step:
        <input
          type="number"
          min={1}
          value={state.step}
          onChange={e => dispatch({ type: 'SET_STEP', payload: Number(e.target.value) })}
        />
      </label>
    </div>
  )
}

Exemplo 2 — Estado de requisição assíncrona

Gerenciar loading, dados e erro com três useState separados é propenso a inconsistências — loading pode ser false enquanto error ainda é o erro anterior. Um reducer com estados nomeados elimina esse problema.

src/hooks/useFetch.tsx 
import { useReducer, useEffect } from 'react'

// ─── Tipos ───────────────────────────────────────────────
// Estado modelado como máquina de estados explícita
type FetchState<T> =
  | { status: 'idle' }
  | { status: 'loading' }
  | { status: 'success'; data: T }
  | { status: 'error'; error: string }

type FetchAction<T> =
  | { type: 'FETCH_STARTED' }
  | { type: 'FETCH_SUCCESS'; payload: T }
  | { type: 'FETCH_ERROR'; payload: string }
  | { type: 'RESET' }

// ─── Reducer ─────────────────────────────────────────────
function fetchReducer<T>(
  _state: FetchState<T>,
  action: FetchAction<T>
): FetchState<T> {
  switch (action.type) {
    case 'FETCH_STARTED':
      return { status: 'loading' }
    case 'FETCH_SUCCESS':
      return { status: 'success', data: action.payload }
    case 'FETCH_ERROR':
      return { status: 'error', error: action.payload }
    case 'RESET':
      return { status: 'idle' }
    default:
      return _state
  }
}

// ─── Hook customizado ────────────────────────────────────
export function useFetch<T>(url: string) {
  const [state, dispatch] = useReducer(
    fetchReducer<T>,
    { status: 'idle' } as FetchState<T>
  )

  useEffect(() => {
    if (!url) return

    dispatch({ type: 'FETCH_STARTED' })

    const controller = new AbortController()

    fetch(url, { signal: controller.signal })
      .then(res => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`)
        return res.json() as Promise<T>
      })
      .then(data => dispatch({ type: 'FETCH_SUCCESS', payload: data }))
      .catch(err => {
        if (err.name === 'AbortError') return
        dispatch({ type: 'FETCH_ERROR', payload: err.message })
      })

    return () => controller.abort()
  }, [url])

  return state
}

// ─── Componente que usa o hook ────────────────────────────
interface Post {
  id: number
  title: string
  body: string
}

export function PostList() {
  const state = useFetch<Post[]>('https://jsonplaceholder.typicode.com/posts?_limit=5')

  if (state.status === 'idle') return <p>Aguardando...</p>
  if (state.status === 'loading') return <p>Carregando...</p>
  if (state.status === 'error') return <p>Erro: {state.error}</p>

  // TypeScript sabe aqui que state.data existe e é Post[]
  return (
    <ul>
      {state.data.map(post => (
        <li key={post.id}><strong>{post.title}</strong></li>
      ))}
    </ul>
  )
}
💡
O poder do narrowing: quando state.status === 'success', TypeScript infere automaticamente que state.data existe. Com três useState separados, você precisaria de verificações manuais. A união discriminada faz isso pelo tipo.

Exemplo 3 — Formulário com validação

Formulários são o caso de uso mais comum para useReducer em aplicações reais. Um formulário com validação sincronizada, campos touched e estado de submissão precisa de transições coordenadas que o reducer modela perfeitamente.

src/components/SignupForm.tsx 
import { useReducer } from 'react'

// ─── Tipos ───────────────────────────────────────────────
interface FormFields {
  name: string
  email: string
  password: string
}

interface FormErrors {
  name?: string
  email?: string
  password?: string
}

interface FormState {
  fields: FormFields
  errors: FormErrors
  touched: Record<keyof FormFields, boolean>
  isSubmitting: boolean
  isSubmitted: boolean
}

type FormAction =
  | { type: 'FIELD_CHANGED'; payload: { field: keyof FormFields; value: string } }
  | { type: 'FIELD_BLURRED'; payload: { field: keyof FormFields } }
  | { type: 'SUBMIT_STARTED' }
  | { type: 'SUBMIT_SUCCESS' }
  | { type: 'SUBMIT_FAILED'; payload: FormErrors }
  | { type: 'RESET' }

// ─── Função de validação (pura, testável separadamente) ──
function validateField(field: keyof FormFields, value: string): string | undefined {
  switch (field) {
    case 'name':
      return value.trim().length < 2 ? 'Nome muito curto' : undefined
    case 'email':
      return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value) ? undefined : 'Email inválido'
    case 'password':
      return value.length < 6 ? 'Mínimo 6 caracteres' : undefined
  }
}

// ─── Reducer ─────────────────────────────────────────────
const initialState: FormState = {
  fields: { name: '', email: '', password: '' },
  errors: {},
  touched: { name: false, email: false, password: false },
  isSubmitting: false,
  isSubmitted: false,
}

function formReducer(state: FormState, action: FormAction): FormState {
  switch (action.type) {

    case 'FIELD_CHANGED': {
      const { field, value } = action.payload
      const error = state.touched[field] ? validateField(field, value) : undefined
      return {
        ...state,
        fields: { ...state.fields, [field]: value },
        errors: { ...state.errors, [field]: error },
      }
    }

    case 'FIELD_BLURRED': {
      const { field } = action.payload
      const error = validateField(field, state.fields[field])
      return {
        ...state,
        touched: { ...state.touched, [field]: true },
        errors: { ...state.errors, [field]: error },
      }
    }

    case 'SUBMIT_STARTED':
      // Valida todos os campos ao submeter
      const allErrors: FormErrors = {}
      ;(Object.keys(state.fields) as Array<keyof FormFields>).forEach(field => {
        const error = validateField(field, state.fields[field])
        if (error) allErrors[field] = error
      })
      return {
        ...state,
        errors: allErrors,
        touched: { name: true, email: true, password: true },
        isSubmitting: Object.keys(allErrors).length === 0,
      }

    case 'SUBMIT_SUCCESS':
      return { ...state, isSubmitting: false, isSubmitted: true }

    case 'SUBMIT_FAILED':
      return { ...state, isSubmitting: false, errors: action.payload }

    case 'RESET':
      return initialState

    default:
      return state
  }
}

// ─── Componente ──────────────────────────────────────────
export function SignupForm() {
  const [state, dispatch] = useReducer(formReducer, initialState)

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault()
    dispatch({ type: 'SUBMIT_STARTED' })

    // Verificar se há erros antes de prosseguir
    const hasErrors = Object.values(state.errors).some(Boolean)
    if (hasErrors) return

    try {
      await new Promise(res => setTimeout(res, 1000)) // Simula API
      dispatch({ type: 'SUBMIT_SUCCESS' })
    } catch {
      dispatch({ type: 'SUBMIT_FAILED', payload: { email: 'Email já cadastrado' } })
    }
  }

  if (state.isSubmitted) {
    return (
      <div>
        <h2>Cadastro realizado!</h2>
        <button onClick={() => dispatch({ type: 'RESET' })}>Novo cadastro</button>
      </div>
    )
  }

  return (
    <form onSubmit={handleSubmit}>
      {(['name', 'email', 'password'] as Array<keyof FormFields>).map(field => (
        <div key={field}>
          <input
            type={field === 'password' ? 'password' : 'text'}
            placeholder={field}
            value={state.fields[field]}
            onChange={e => dispatch({
              type: 'FIELD_CHANGED',
              payload: { field, value: e.target.value }
            })}
            onBlur={() => dispatch({ type: 'FIELD_BLURRED', payload: { field } })}
          />
          {state.touched[field] && state.errors[field] && (
            <span style={{ color: 'red' }}>{state.errors[field]}</span>
          )}
        </div>
      ))}

      <button type="submit" disabled={state.isSubmitting}>
        {state.isSubmitting ? 'Enviando...' : 'Cadastrar'}
      </button>
    </form>
  )
}

Context + Reducer — o padrão poderoso

Context distribui dados pela árvore. Reducer gerencia a lógica de transição de estado. Combinados, formam o padrão que cobre 90% dos casos onde você pensaria em usar Redux em um projeto novo.

A ideia: o Context expõe o estado atual e a função dispatch. Qualquer componente pode ler o estado e disparar ações — sem passar callbacks por props, sem estados duplicados.

src/contexts/TaskContext.tsx — padrão completo 
import { createContext, useContext, useReducer } from 'react'

// ─── Tipos ───────────────────────────────────────────────
interface Task {
  id: string
  title: string
  done: boolean
}

interface TaskState {
  tasks: Task[]
}

type TaskAction =
  | { type: 'TASK_ADDED'; payload: string }
  | { type: 'TASK_TOGGLED'; payload: string }
  | { type: 'TASK_REMOVED'; payload: string }

// ─── Reducer ─────────────────────────────────────────────
function taskReducer(state: TaskState, action: TaskAction): TaskState {
  switch (action.type) {
    case 'TASK_ADDED':
      return {
        tasks: [...state.tasks, {
          id: crypto.randomUUID(),
          title: action.payload,
          done: false
        }]
      }
    case 'TASK_TOGGLED':
      return {
        tasks: state.tasks.map(t =>
          t.id === action.payload ? { ...t, done: !t.done } : t
        )
      }
    case 'TASK_REMOVED':
      return { tasks: state.tasks.filter(t => t.id !== action.payload) }
    default:
      return state
  }
}

// ─── Contextos separados para estado e dispatch ───────────
// Separar evita que componentes que só despacham re-renderizem
// quando o estado muda
const TaskStateContext = createContext<TaskState | null>(null)
const TaskDispatchContext = createContext<React.Dispatch<TaskAction> | null>(null)

// ─── Provider ────────────────────────────────────────────
export function TaskProvider({ children }: { children: React.ReactNode }) {
  const [state, dispatch] = useReducer(taskReducer, { tasks: [] })

  return (
    <TaskStateContext.Provider value={state}>
      <TaskDispatchContext.Provider value={dispatch}>
        {children}
      </TaskDispatchContext.Provider>
    </TaskStateContext.Provider>
  )
}

// ─── Hooks públicos ──────────────────────────────────────
export function useTaskState(): TaskState {
  const ctx = useContext(TaskStateContext)
  if (!ctx) throw new Error('useTaskState fora do TaskProvider')
  return ctx
}

export function useTaskDispatch(): React.Dispatch<TaskAction> {
  const ctx = useContext(TaskDispatchContext)
  if (!ctx) throw new Error('useTaskDispatch fora do TaskProvider')
  return ctx
}

// ─── Componentes consumidores ────────────────────────────

// Esse componente só re-renderiza quando o estado muda
export function TaskCounter() {
  const { tasks } = useTaskState()
  const done = tasks.filter(t => t.done).length
  return <span>{done}/{tasks.length} concluídas</span>
}

// Esse componente nunca re-renderiza por mudança de estado
// (dispatch é estável — não causa re-render)
export function AddTaskForm() {
  const dispatch = useTaskDispatch()
  const [title, setTitle] = React.useState('')

  const submit = (e: React.FormEvent) => {
    e.preventDefault()
    if (!title.trim()) return
    dispatch({ type: 'TASK_ADDED', payload: title.trim() })
    setTitle('')
  }

  return (
    <form onSubmit={submit}>
      <input value={title} onChange={e => setTitle(e.target.value)} />
      <button type="submit">Adicionar</button>
    </form>
  )
}
🔍
Dois contextos separados: dividir em StateContext e DispatchContext é uma otimização real. O dispatch do useReducer é sempre estável (mesma referência entre renders), então componentes que só despacham ações nunca re-renderizam por mudança de estado.
09

Boas práticas

✓
Reducer fora do componente — garante que seja estável, testável e não cause re-renders desnecessários.
✓
Nunca mute o estado — sempre retorne um novo objeto com spread. Mutação direta não dispara re-render e causa bugs silenciosos.
✓
Action types em UPPER_SNAKE_CASE — convenção universal, facilita leitura de logs e DevTools.
✓
Nome de action no passado — TASK_ADDED, não ADD_TASK. A action descreve o que aconteceu, não a intenção.
⚠️
Nunca dispare efeitos colaterais no reducer — sem fetch, sem timers, sem localStorage. O reducer é puro. Efeitos ficam em useEffect ou nos handlers dos componentes.
⚠️
Immer para estado muito aninhado — useImmerReducer (lib immer) permite mutação aparente com resultado imutável. Útil quando o spread fica profundo demais.

Básico Exercício 1 — Semáforo

Um semáforo é o exemplo clássico de máquina de estados. Tem um estado atual (a cor) e uma transição definida (avançar para a próxima cor). Este exercício força o pensamento em termos de estados discretos e transições nomeadas.

  • O estado deve ser um objeto com color: 'red' | 'yellow' | 'green'
  • Crie uma action NEXT que avança a cor: red → green → yellow → red
  • O componente exibe um círculo colorido e um botão "Avançar"
  • Bônus: adicione uma action AUTO_TICK e um useEffect com setInterval de 2000ms que faz o semáforo avançar automaticamente
// TrafficLight.tsx
Solução — TrafficLight.tsx 
import { useReducer, useEffect } from 'react'

type Color = 'red' | 'yellow' | 'green'

interface TrafficState { color: Color }
type TrafficAction = { type: 'NEXT' }

// Tabela de transição — explícita e auditável
const transitions: Record<Color, Color> = {
  red: 'green',
  green: 'yellow',
  yellow: 'red',
}

const colorMap: Record<Color, string> = {
  red: '#f87171',
  yellow: '#fbbf24',
  green: '#34d399',
}

const labelMap: Record<Color, string> = {
  red: '🔴 Pare',
  yellow: '🟡 Atenção',
  green: '🟢 Siga',
}

function trafficReducer(state: TrafficState, action: TrafficAction): TrafficState {
  switch (action.type) {
    case 'NEXT':
      return { color: transitions[state.color] }
    default:
      return state
  }
}

export function TrafficLight() {
  const [state, dispatch] = useReducer(trafficReducer, { color: 'red' })

  // Bônus: avanço automático
  useEffect(() => {
    const timer = setInterval(() => {
      dispatch({ type: 'NEXT' })
    }, 2000)
    return () => clearInterval(timer)
  }, []) // dispatch é estável, não precisa de deps

  return (
    <div style={{ textAlign: 'center' }}>
      <div style={{
        width: 80,
        height: 80,
        borderRadius: '50%',
        background: colorMap[state.color],
        margin: '0 auto 16px',
        transition: 'background 0.3s',
      }} />
      <p>{labelMap[state.color]}</p>
      <button onClick={() => dispatch({ type: 'NEXT' })}>
        Avançar
      </button>
    </div>
  )
}
Médio Exercício 2 — Carrinho estilo Redux

Reimplemente o carrinho de compras do módulo de Context usando useReducer. O objetivo é comparar as duas abordagens e sentir quando cada uma é mais adequada.

  • Modele o estado com items: CartItem[] e isOpen: boolean
  • Actions: ITEM_ADDED, ITEM_REMOVED, QUANTITY_CHANGED, CART_CLEARED, CART_TOGGLED
  • Calcule totalItems e totalPrice derivados do estado (não no reducer — no componente)
  • O reducer deve lidar com a lógica de incremento quando o item já existe em ITEM_ADDED
  • Bônus: combine com Context para tornar o estado global (padrão Context + Reducer)
// cartReducer.ts
Solução — cartReducer.ts 
// ─── Tipos ───────────────────────────────────────────────
interface CartItem {
  id: string
  name: string
  price: number
  quantity: number
}

interface CartState {
  items: CartItem[]
  isOpen: boolean
}

type CartAction =
  | { type: 'ITEM_ADDED'; payload: Omit<CartItem, 'quantity'> }
  | { type: 'ITEM_REMOVED'; payload: { id: string } }
  | { type: 'QUANTITY_CHANGED'; payload: { id: string; quantity: number } }
  | { type: 'CART_CLEARED' }
  | { type: 'CART_TOGGLED' }

// ─── Estado inicial ──────────────────────────────────────
export const cartInitialState: CartState = {
  items: [],
  isOpen: false,
}

// ─── Reducer ─────────────────────────────────────────────
export function cartReducer(state: CartState, action: CartAction): CartState {
  switch (action.type) {

    case 'ITEM_ADDED': {
      const exists = state.items.find(i => i.id === action.payload.id)
      if (exists) {
        return {
          ...state,
          items: state.items.map(i =>
            i.id === action.payload.id
              ? { ...i, quantity: i.quantity + 1 }
              : i
          ),
        }
      }
      return {
        ...state,
        items: [...state.items, { ...action.payload, quantity: 1 }],
      }
    }

    case 'ITEM_REMOVED':
      return {
        ...state,
        items: state.items.filter(i => i.id !== action.payload.id),
      }

    case 'QUANTITY_CHANGED':
      if (action.payload.quantity <= 0) {
        return {
          ...state,
          items: state.items.filter(i => i.id !== action.payload.id),
        }
      }
      return {
        ...state,
        items: state.items.map(i =>
          i.id === action.payload.id
            ? { ...i, quantity: action.payload.quantity }
            : i
        ),
      }

    case 'CART_CLEARED':
      return { ...state, items: [] }

    case 'CART_TOGGLED':
      return { ...state, isOpen: !state.isOpen }

    default:
      return state
  }
}

// ─── Seletores (valores derivados) ───────────────────────
// Calculados no componente ou no hook, não no reducer
export const selectTotalItems = (state: CartState) =>
  state.items.reduce((sum, i) => sum + i.quantity, 0)

export const selectTotalPrice = (state: CartState) =>
  state.items.reduce((sum, i) => sum + i.price * i.quantity, 0)
Intermediário Exercício 3 — Wizard de cadastro multi-etapa

Um wizard (formulário de múltiplas etapas) é uma máquina de estados sofisticada. O reducer gerencia qual etapa está ativa, os dados acumulados de cada etapa e o estado geral do processo. Este é o exercício que mais se aproxima de código de produção.

  • O wizard tem 3 etapas: personal (nome, email), address (cidade, estado), review (confirmação)
  • O estado guarda a etapa atual, os dados de cada etapa e se o processo foi concluído
  • Actions: STEP_COMPLETED (avança e salva dados), STEP_BACK, WIZARD_SUBMITTED, WIZARD_RESET
  • Na etapa review, exibe todos os dados coletados antes da submissão
  • Desafio: adicione validação por etapa — STEP_COMPLETED só avança se os dados da etapa forem válidos
// wizardReducer.ts
Solução — wizardReducer.ts 
// ─── Tipos ───────────────────────────────────────────────
type Step = 'personal' | 'address' | 'review'

interface PersonalData { name: string; email: string }
interface AddressData { city: string; state: string }

interface WizardState {
  currentStep: Step
  data: {
    personal: PersonalData | null
    address: AddressData | null
  }
  isCompleted: boolean
}

type WizardAction =
  | { type: 'STEP_COMPLETED'; payload: { step: 'personal'; data: PersonalData } }
  | { type: 'STEP_COMPLETED'; payload: { step: 'address'; data: AddressData } }
  | { type: 'STEP_BACK' }
  | { type: 'WIZARD_SUBMITTED' }
  | { type: 'WIZARD_RESET' }

// ─── Sequência de etapas ─────────────────────────────────
const stepOrder: Step[] = ['personal', 'address', 'review']

function getNextStep(current: Step): Step {
  const idx = stepOrder.indexOf(current)
  return stepOrder[Math.min(idx + 1, stepOrder.length - 1)]
}

function getPrevStep(current: Step): Step {
  const idx = stepOrder.indexOf(current)
  return stepOrder[Math.max(idx - 1, 0)]
}

// ─── Estado inicial ──────────────────────────────────────
const initialState: WizardState = {
  currentStep: 'personal',
  data: { personal: null, address: null },
  isCompleted: false,
}

// ─── Reducer ─────────────────────────────────────────────
function wizardReducer(state: WizardState, action: WizardAction): WizardState {
  switch (action.type) {

    case 'STEP_COMPLETED':
      if (action.payload.step === 'personal') {
        return {
          ...state,
          currentStep: getNextStep(state.currentStep),
          data: { ...state.data, personal: action.payload.data },
        }
      }
      if (action.payload.step === 'address') {
        return {
          ...state,
          currentStep: getNextStep(state.currentStep),
          data: { ...state.data, address: action.payload.data },
        }
      }
      return state

    case 'STEP_BACK':
      return { ...state, currentStep: getPrevStep(state.currentStep) }

    case 'WIZARD_SUBMITTED':
      return { ...state, isCompleted: true }

    case 'WIZARD_RESET':
      return initialState

    default:
      return state
  }
}

// ─── Componente Wizard ───────────────────────────────────
import { useReducer, useState } from 'react'

export function Wizard() {
  const [state, dispatch] = useReducer(wizardReducer, initialState)

  if (state.isCompleted) {
    return (
      <div>
        <h2>✓ Cadastro concluído!</h2>
        <button onClick={() => dispatch({ type: 'WIZARD_RESET' })}>Recomeçar</button>
      </div>
    )
  }

  return (
    <div>
      <p>Etapa: {stepOrder.indexOf(state.currentStep) + 1} de {stepOrder.length}</p>

      {state.currentStep === 'personal' && (
        <PersonalStep
          onNext={(data) => dispatch({ type: 'STEP_COMPLETED', payload: { step: 'personal', data } })}
        />
      )}

      {state.currentStep === 'address' && (
        <AddressStep
          onBack={() => dispatch({ type: 'STEP_BACK' })}
          onNext={(data) => dispatch({ type: 'STEP_COMPLETED', payload: { step: 'address', data } })}
        />
      )}

      {state.currentStep === 'review' && state.data.personal && state.data.address && (
        <ReviewStep
          personal={state.data.personal}
          address={state.data.address}
          onBack={() => dispatch({ type: 'STEP_BACK' })}
          onSubmit={() => dispatch({ type: 'WIZARD_SUBMITTED' })}
        />
      )}
    </div>
  )
}

// ─── Subcomponentes das etapas ───────────────────────────
function PersonalStep({ onNext }: { onNext: (d: PersonalData) => void }) {
  const [name, setName] = useState('')
  const [email, setEmail] = useState('')
  return (
    <div>
      <h3>Dados pessoais</h3>
      <input placeholder="Nome" value={name} onChange={e => setName(e.target.value)} />
      <input placeholder="Email" value={email} onChange={e => setEmail(e.target.value)} />
      <button onClick={() => onNext({ name, email })} disabled={!name || !email}>
        Próximo →
      </button>
    </div>
  )
}

function AddressStep({ onBack, onNext }: { onBack: () => void; onNext: (d: AddressData) => void }) {
  const [city, setCity] = useState('')
  const [stateVal, setStateVal] = useState('')
  return (
    <div>
      <h3>Endereço</h3>
      <input placeholder="Cidade" value={city} onChange={e => setCity(e.target.value)} />
      <input placeholder="Estado" value={stateVal} onChange={e => setStateVal(e.target.value)} />
      <button onClick={onBack}>← Voltar</button>
      <button onClick={() => onNext({ city, state: stateVal })} disabled={!city || !stateVal}>
        Próximo →
      </button>
    </div>
  )
}

function ReviewStep({ personal, address, onBack, onSubmit }: {
  personal: PersonalData; address: AddressData;
  onBack: () => void; onSubmit: () => void
}) {
  return (
    <div>
      <h3>Revisão</h3>
      <p><strong>Nome:</strong> {personal.name}</p>
      <p><strong>Email:</strong> {personal.email}</p>
      <p><strong>Cidade:</strong> {address.city}</p>
      <p><strong>Estado:</strong> {address.state}</p>
      <button onClick={onBack}>← Voltar</button>
      <button onClick={onSubmit}>✓ Confirmar</button>
    </div>
  )
}
